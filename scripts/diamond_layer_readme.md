# Diamond Layer: The Gauntlet & Final Analysis

The Diamond Layer is the final and most critical stage of the strategy discovery pipeline. It acts as the ultimate quality gate, taking the promising strategies discovered by the Platinum Layer and subjecting them to a rigorous gauntlet of high-fidelity backtesting, performance filtering, and cross-market validation.

Its purpose is to move beyond theoretical potential and answer the definitive question: **"Does this strategy actually make money under realistic market conditions, and is it robust enough to work on more than just the data it was discovered on?"**

The layer is composed of a shared simulation engine and three distinct scripts that form a cohesive workflow:

1.  **`simulation_engine.py`**: A centralized, high-fidelity backtesting engine used by both the backtester and validator to ensure 100% consistent simulation logic.
2.  **`diamond_data_prepper.py`**: A foundational script that prepares all necessary data and pre-computes the exact timestamps ("triggers") where each strategy's entry conditions were met.
3.  **`diamond_backtester.py`**: The first quality gate. It performs an in-sample backtest to filter for a small, elite subset of "Master Strategies".
4.  **`diamond_validator.py`**: The final gauntlet. It performs out-of-sample and cross-market validation on the Master Strategies and generates a suite of comprehensive analytical reports.

---

## Part 1: `diamond_data_prepper.py` - The Trigger Engine

### Core Purpose

This script embodies the "Prepare Once, Test Many" philosophy of the pipeline. Its core function is to perform the heavy, one-time computation of finding **every single timestamp where a strategy's market conditions were met**. By pre-calculating these "trigger times" and saving them to small, efficient files, the subsequent backtesting scripts can run incredibly fast, as they no longer need to re-evaluate the strategy rules on every candle.

### How It Works

1.  **Cross-Market Data Preparation**: Before doing anything else, the script identifies the timeframe of the master instrument (e.g., `60m` from `EURUSD60`). It then scans the `raw_data` directory and ensures that the Silver (features) and Gold (ML-ready) data files have been generated for **all** instruments sharing that same timeframe. It intelligently invokes the generator scripts from previous layers only when needed.
2.  **New Strategy Identification**: It compares the list of discovered strategies from the Platinum layer against a master list of strategies it has already processed for the Diamond layer. This allows it to identify only the new strategies that need their triggers extracted.
3.  **Parallel Trigger Extraction**: The script loads the Gold features for a given market. It then iterates through each new strategy and uses Pandas' high-performance `.query()` method to find all the rows (and thus, timestamps) that match the strategy's `market_rule`.
4.  **Organized Output**: The resulting trigger timestamps are saved to a highly organized directory structure: `diamond_data/triggers/{master_instrument}/{market_to_scan}/{trigger_key}.parquet`. This structure makes it trivial for the backtester and validator to find the exact triggers they need for any given strategy/market combination.

## Part 2: `diamond_backtester.py` - The Mastery Engine

### Core Purpose

This script is the first and most important quality filter in the Diamond Layer. Its sole purpose is to perform a high-fidelity, parallelized backtest of every discovered strategy **on its home instrument only**. This in-sample test quickly filters the vast universe of potential strategies down to a small, elite subset of "Master Strategies" that demonstrate robust performance on the data they were discovered on.

### How It Works

1.  **Data Loading**: The script loads the strategies to be tested and the Silver features for the master instrument.
2.  **High-Fidelity Simulation**: Using the shared `simulation_engine.py`, it simulates each strategy in parallel:
    - It fetches the pre-computed trigger times from the files generated by the prepper.
    - For each trigger, it simulates the trade on a candle-by-candle basis, looking forward in time to see if the SL or TP is hit.
    - The simulation accurately models costs, including **spread** and **commission**, as defined in `config.py`.
3.  **Comprehensive Metrics Calculation**: After simulating all trades for a strategy, it calculates a wide array of professional performance metrics, including:
    - Profit Factor
    - Win Rate %
    - Max Drawdown
    - System Quality Number (SQN)
    - Total Trades
    - Average Win/Loss
4.  **Quality Filtering**: It applies a strict set of criteria from `config.py` (e.g., `DIAMOND_MIN_PROFIT_FACTOR`, `DIAMOND_MAX_DRAWDOWN_PCT`) to the results. Strategies that pass all criteria are promoted to "Master Strategy" status.
5.  **Feedback Loop Generation**: Strategies that _fail_ the quality check are added to a **blacklist** file (`platinum_data/blacklists/{instrument}.parquet`). This file provides the critical feedback to the Platinum Layer, allowing it to learn from its mistakes and refine future discovery runs.

### Inputs and Outputs (Backtester)

- **Input**: `diamond_data/strategies/{instrument}.parquet`, `diamond_data/triggers/`, and `silver_data/features/{instrument}.parquet`.
- **Outputs**:
  - `diamond_data/master_reports/{instrument}.parquet`: The list of strategies that passed the filter, along with their in-sample performance metrics. This is the input for the final validation stage.
  - `platinum_data/blacklists/{instrument}.parquet`: The feedback file containing failed strategies.

## Part 3: `diamond_validator.py` - The Gauntlet & Analyser

### Core Purpose

This is the final script in the entire pipeline. It takes the elite "Master Strategies" identified by the backtester and subjects them to the final gauntlet: a comprehensive stress test across a portfolio of related instruments. This step validates the robustness of a strategy and provides the rich, detailed data needed for final analysis.

### How It Works

1.  **Loading Master Strategies**: The script loads the list of master strategies from the backtester's report.
2.  **Cross-Market Validation**: It identifies all available instruments that share the same timeframe as the master instrument (including the master instrument itself for a complete picture). It then iterates through each of these "validation markets."
3.  **Parallel Simulation**: For each market, it runs a high-fidelity simulation for every master strategy, again using the shared `simulation_engine`. This time, it requests a "deep log," which saves a rich, detailed record for every single trade.
4.  **Final Report Generation**: After all simulations are complete, it aggregates the millions of trade records to generate the final suite of analytical reports:
    - **Raw Trade Logs**: Every single trade from every strategy on every market is saved to `diamond_data/trade_logs/`. This provides ultimate transparency and allows for deep offline analysis.
    - **Detailed Report**: Performance metrics for each strategy on _every individual market_ it was tested on. This helps identify which strategies are specialists and which are generalists.
    - **Summary Report**: The _average_ performance metrics for each strategy across all tested markets, giving a high-level view of its overall robustness.
    - **Regime Analysis Report**: A powerful breakdown of strategy performance under different market conditions (e.g., 'trending' vs. 'ranging', 'high-vol' vs. 'low-vol', different trading sessions). This helps to understand _when_ and _why_ a strategy works.

### Inputs and Outputs (Validator)

- **Input**: `diamond_data/master_reports/{instrument}.parquet`.
- **Outputs**: The final, valuable outputs of the entire pipeline, located in `diamond_data/final_reports/` and `diamond_data/trade_logs/`.

## Configuration

The Diamond Layer's behavior is controlled by simulation and filtering parameters in `config.py`:

- **Simulation & Cost Model**:
  - `SIMULATION_SPREAD_PIPS`: The assumed spread for different instruments.
  - `SIMULATION_COMMISSION_PER_LOT`: The round-trip commission cost.
  - `SIMULATION_MAX_LOOKFORWARD`: The maximum number of candles a trade will be held.
- **Performance Filters**:
  - `DIAMOND_MIN_PROFIT_FACTOR`
  - `DIAMOND_MAX_DRAWDOWN_PCT`
  - `DIAMOND_MIN_TOTAL_TRADES`
  - `DIAMOND_MIN_SQN`

## How to Run the Scripts

The three Diamond Layer scripts must be run in sequence.

1.  **Via the Master Orchestrator (Recommended)**:
    The `orchestrator.py` will run all three scripts in the correct order automatically.
    ```bash
    python orchestrator.py
    ```
2.  **Standalone (Targeted Mode)**:
    You must run the scripts in sequence, targeting the same instrument name.

    ```bash
    # Step 1: Prepare the data and extract triggers
    python scripts/diamond_data_prepper.py EURUSD60

    # Step 2: Run the in-sample backtest to find Master Strategies
    python scripts/diamond_backtester.py EURUSD60

    # Step 3: Run the cross-market validation and generate final reports
    python scripts/diamond_validator.py EURUSD60
    ```
